## Enhancement 1: Sticky-scroll — respect user scrollback during streaming (Impact:5, Effort:1)

### Problem
`handle_session_event` in `src/tui/app/session_events.rs` calls `app.state.scroll_to_bottom()` on EVERY event (line ~20 and at least 7 more places). If the user scrolls up to read prior output while the agent streams, the next text chunk yanks them back to the bottom. This is the #1 daily papercut.

### Fix
1. Add `pub chat_auto_follow: bool` field to `AppState` in `src/tui/app/state/mod.rs`, default `true`
2. In `src/tui/app/state/scroll.rs`:
   - `scroll_up()` should set `chat_auto_follow = false` before doing its work
   - In the existing `scroll_up`, add `self.chat_auto_follow = false;` at the start
   - `scroll_to_bottom()` should set `chat_auto_follow = true`
3. In `src/tui/app/session_events.rs`:
   - Replace every `app.state.scroll_to_bottom()` call with:
     ```rust
     if app.state.chat_auto_follow { app.state.scroll_to_bottom(); }
     ```
   - There are ~8 occurrences. The very first one at line ~20 is the unconditional scroll; keep that as just the guarded version.
   - But for `ToolCallStart` and `ToolCallComplete`, ALWAYS scroll (tool calls should force-follow since we added auto-follow to tool panel). So for those two event variants, keep unguarded `app.state.scroll_to_bottom()`.

## Enhancement 3: Slash-command argument hints (Impact:4, Effort:2)

### Problem
In `src/tui/app/state/slash_suggest.rs`, `slash_suggestions_visible()` returns `false` once whitespace is typed (`!self.input.trim().chars().any(char::is_whitespace)`). So after `/spawn `, suggestions vanish and the user has no hint about `<name> [instructions]`.

### Fix

1. Create `src/tui/app/state/slash_hints.rs` — a new module providing usage strings:

```rust
//! Usage hint strings for slash commands.

/// Returns a usage hint for the given slash command, or None.
pub fn usage_hint(cmd: &str) -> Option<&'static str> {
    match cmd {
        "/spawn" => Some("/spawn <name> [instructions]  — create a sub-agent"),
        "/kill" => Some("/kill <name>  — terminate a sub-agent"),
        "/agent" => Some("/agent <name> [msg]  — focus or message a sub-agent"),
        "/talk" => Some("/talk <name> <msg>  — send to a sub-agent"),
        "/add" => Some("/add <name>  — alias for /spawn"),
        "/ask" => Some("/ask <question>  — ephemeral side question"),
        "/file" => Some("/file <path>  — attach a file"),
        "/image" => Some("/image <path>  — attach an image"),
        "/autochat" => Some("/autochat [count] <task>  — multi-agent relay"),
        "/ralph" => Some("/ralph <subcommand>  — run/submit/status"),
        "/model" => Some("/model <name>  — switch LLM model"),
        "/focus" => Some("/focus <agent>  — switch to agent's chat"),
        "/ls" | "/list" => Some("/ls  — list spawned agents"),
        "/rm" | "/remove" => Some("/rm <name>  — remove an agent"),
        _ => None,
    }
}
```

2. In `src/tui/app/state/mod.rs`, add `pub mod slash_hints;`

3. In `src/tui/app/state/slash_suggest.rs`:
   - Change `slash_suggestions_visible()` to also show when there's a hint:
     ```rust
     pub fn slash_suggestions_visible(&self) -> bool {
         self.view_mode == crate::tui::models::ViewMode::Chat
             && !self.processing
             && self.input.starts_with('/')
     }
     ```
     Remove the whitespace check — suggestions should stay visible while typing args.
   
   - Add a new method:
     ```rust
     pub fn current_slash_hint(&self) -> Option<&'static str> {
         let cmd = self.input.split_whitespace().next()?;
         super::slash_hints::usage_hint(cmd)
     }
     ```

4. In `src/tui/ui/chat_view/suggestions.rs`, after the suggestion list rendering, add a hint line render:
   After the list is rendered, if `slash_suggestions` is empty but there's a hint, render just a single-line hint:
   
   At the end of `render_suggestions`, add:
   ```rust
   // Show usage hint when suggestions list is empty but we have args typed
   if app.state.slash_suggestions.is_empty() {
       if let Some(hint) = app.state.current_slash_hint() {
           let hint_item = ListItem::new(Line::from(vec![
               Span::raw("  "),
               Span::styled(hint.to_string(), Style::default().fg(Color::DarkGray).dim()),
           ]));
           let hint_list = List::new(vec![hint_item])
               .block(Block::default().borders(Borders::ALL).title(" Usage "));
           f.render_stateful_widget(hint_list, area, &mut list_state);
       }
   }
   ```

## Enhancement 4: Context-budget feedback UI (Impact:4, Effort:2)

### Problem
Session events like `TokenEstimate`, `Compaction*`, and `ContextTruncated` are silently dropped. Users have no visibility into context usage or when compaction happens.

### Fix

1. First, read `src/session/mod.rs` or search for `SessionEvent` enum to find the exact variant names for compaction/token events.

2. In `src/tui/app/session_events.rs`, add handling for the token/compaction events that currently fall into the catch-all `_ => {}` arm. Add BEFORE the catch-all:

```rust
SessionEvent::TokenEstimate { used, budget } => {
    app.state.context_used = Some(*used);
    app.state.context_budget = Some(*budget);
}
SessionEvent::ContextTruncated => {
    app.state.messages.push(ChatMessage::new(
        MessageType::System,
        "⚠ Context was truncated to fit budget".to_string(),
    ));
}
SessionEvent::CompactionStart => {
    app.state.status = "Compacting context…".to_string();
}
SessionEvent::CompactionComplete { original, compacted } => {
    app.state.messages.push(ChatMessage::new(
        MessageType::System,
        format!("Context compacted: {original} → {compacted} messages"),
    ));
    app.state.status = "Ready".to_string();
}
```

IMPORTANT: Read the actual SessionEvent enum definition first! The variant names above may not match exactly. Search for the actual enum and use the correct field names. If the variants don't exist, skip this enhancement.

3. In `src/tui/app/state/mod.rs`, add these fields to AppState (in the existing field list, don't create duplicate fields — search first):
```rust
pub context_used: Option<usize>,
pub context_budget: Option<usize>,
```

4. In `src/tui/ui/chat_view/title.rs` or `src/tui/ui/chat_view/token_spans.rs`, add a context usage span. Read these files first. If there's already a token/usage display, add a context bar next to it:

Look for where token spans are rendered and add:
```rust

    let pct = ((*used as f64 / *budget as f64) * 100.0) as usize;

```
