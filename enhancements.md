## What you have today (baseline)                                                                                                                                                                                           │
│                                                                                                                                                                                                                              │
│  - Policies: Legacy (keep_last cascade 16→12→8→6→3→1 → terminal truncation), Reset (Lu et al., 32k threshold), Incremental (select-then-pack with lexical relevance scoring), OracleReplay                                   │
│  - Summary index with Granularity::{Turn, Phase, Session} but only turn-level production wired                                                                                                                               │
│  - Background RLM summarization — for large tool outputs only                                                                                                                                                                │
│  - Recall escape hatches: session_recall, context_browse, context_pin                                                                                                                                                        │
│                                                                                                                                                                                                                              │
│  ## Highest-leverage improvements                                                                                                                                                                                            │
│                                                                                                                                                                                                                              │
│  ### 1. Structured state object, not prose summaries (biggest win)                                                                                                                                                           │
│  The reset/compaction summary is freeform prose. Long-horizon failure mode #1 is losing constraints and goal state, not facts. Make every derived context carry a typed, always-present "state header" rebuilt               │
│  deterministically from the session:                                                                                                                                                                                         │
│                                                                                                                                                                                                                              │
│  - goal + success criteria (session_task ledger already exists)                                                                                                                                                              │
│  - open todos / blocked items                                                                                                                                                                                                │
│  - pinned turns (context_pin exists but verify pins are actually injected into derive_incremental and derive_reset — incremental's scorer in incremental.rs has no pin term at all)                                          │
│  - decisions made + forbidden actions                                                                                                                                                                                        │
│  - evidence/proof IDs                                                                                                                                                                                                        │
│                                                                                                                                                                                                                              │
│  This converts compression from "hope the summarizer kept it" into a lossless channel for the things that actually matter at horizon 100+ turns.                                                                             │
│                                                                                                                                                                                                                              │
│  ### 2. Calibrated token estimation with provider feedback                                                                                                                                                                   │
│  token.rs delegates everything to RlmChunker::estimate_tokens (heuristic) and images are a flat 2000. Estimation error compounds: over-estimate → premature compression (semantic loss); under-estimate → 400 errors →       │
│  retry cascade → more aggressive compression. Fix:                                                                                                                                                                           │
│                                                                                                                                                                                                                              │
│  - Record actual prompt_tokens from provider usage on every response, compute a per-model correction ratio (EWMA), and apply it online. Cheap, no tokenizer dependency, converges in a few turns.                            │
│  - Model-specific image costs.                                                                                                                                                                                               │
│                                                                                                                                                                                                                              │
│  ### 3. Finish the hierarchical index (Phase/Session rollups)                                                                                                                                                                │
│  Granularity::Phase/Session exist in index/types.rs but index_produce only does turn-level. Implement RAPTOR-style rollups: turn summaries → phase summaries → session summary, with generation-based invalidation           │
│  (already in SummaryNode.generation). Then derive_incremental's gap-filling can pick granularity by gap size — a 200-message gap gets one phase summary, not nothing or one flat blob. This is the difference between        │
│  O(n) and O(log n) context cost as sessions grow.                                                                                                                                                                            │
│                                                                                                                                                                                                                              │
│  ### 4. Write-time salience, not read-time summarization                                                                                                                                                                     │
│  Background RLM (rlm_background/) only handles tool outputs. Extend the same spawn/cache/route pattern to turns: when a turn is appended, a background job tags it with salience score, topic tags, and a 1-line gist.       │
│  Then derivation becomes pure selection (fast, deterministic, no hot-path LLM call). Today enforce_context_window can block the agent step on summarization.                                                                 │
│                                                                                                                                                                                                                              │
│  Related: compaction triggers at 90% of budget — reactive. Trigger background pre-compaction at ~60% so the hot path never pays the latency.                                                                                 │
│                                                                                                                                                                                                                              │
│  ### 5. Upgrade the incremental scorer                                                                                                                                                                                       │
│  incremental.rs is hand-tuned lexical overlap (file=4.0, tool=2.5, error=3.0, recency=0.05). Limitations:                                                                                                                    │
│                                                                                                                                                                                                                              │
│  - Task signature comes only from the most recent user turn's RelevanceMeta — a "continue" turn produces an empty signature and selection degrades to pure recency (the reset policy handles this via active_user_tail_st    │
│  art; incremental doesn't).                                                                                                                                                                                                  │
│  - No semantic match: a turn about "auth middleware" won't match a task about "bearer token validation". Cache per-turn embeddings in the index and add a cosine term, or use the write-time gists from #4.                  │
│  - No pin/goal term (see #1).                                                                                                                                                                                                │
│                                                                                                                                                                                                                              │
│  ### 6. Make dropped context navigable, not silent                                                                                                                                                                           │
│  dropped_ranges is tracked but the model just sees [AUTO CONTEXT COMPRESSION] + advice to call session_recall. Render the dropped ranges as a compact table of contents in the compression marker: turns 12–47:              │
│  implemented auth middleware (call context_browse show_turn N for detail). This turns recall from blind search into indexed lookup — the model knows what it forgot and where to find it.                                    │
│                                                                                                                                                                                                                              │
│  ### 7. Model-relative thresholds                                                                                                                                                                                            │
│  DEFAULT_RESET_THRESHOLD_TOKENS = 32_000 is absolute. On a 200k-context model you're compressing at 16% utilization; on a 32k model you're compressing at 100% (too late). Make thresholds a fraction of context_window_f    │
│  or_model with env override.                                                                                                                                                                                                 │
│                                                                                                                                                                                                                              │
│  ### 8. RLM recursion improvements                                                                                                                                                                                           │
│  - repl.rs: structured tool-call dispatch is gated behind functiongemma; the regex-parsed DSL fallback is fragile for deep recursion. Make structured dispatch the default path.                                             │
│  - Add sub-query result caching keyed on (chunk hash, query) — recursive analyses over a slowly-growing transcript re-summarize mostly identical chunks every time.                                                          │
│                                                                                                                                                                                                                              │
│  ### 9. Close the loop: recall-fidelity evals                                                                                                                                                                                │
│  src/session/eval/ exists. Add a needle-in-haystack harness: inject facts at turn N, run M turns of filler, compress, ask the model to retrieve. Score each policy/threshold combo. Without this metric, all tuning of       │
│  #1–#7 is blind. This is the instrument that lets you claim 100x rather than hope for it.                                                                                                                                    │
